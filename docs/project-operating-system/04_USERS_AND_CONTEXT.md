# Users and Context

## Primary User

### Role

Builder, developer, or AI workflow designer.

### Situation

They need a specialist agent or coordinated agent workflow for a concrete project task.

### Pain

Generic prompts are too broad, and large agent libraries become hard to browse manually.

### Goal

Find the right agent quickly, inspect the source, and use it in their preferred AI coding or productivity tool.

### Skill Level

Technical or semi-technical. Comfortable with Markdown, local scripts, and browser-based tools.

### Environment

Local machine first. Optional browser UI. Optional API key for Gemini-enhanced matching.

### What They Do Not Want

Mandatory cloud setup, broken local usage, unclear provenance, or broad rewrites that make the content harder to use.

## Secondary User

### Role

Maintainer or contributor.

### Goal

Add agents, improve tooling, import sources, or export packs without breaking existing workflows.

## Accessibility and Usability Needs

- Search and filtering should remain clear.
- Static usage should work without an API key.
- Setup docs should distinguish local-only behavior from optional external services.

## Unknowns

- Production web audience for `supabase-web`.
- Whether non-technical users are a primary target.
